B. One Eye, One Edge

Meet the eye 🔦

A flashlight + light meter in one brick — it measures its own echo.

left_eye.reflection()   # 0-100: % of its OWN light echoing back
left_eye.color()        # a name: Color.RED... (Week 4's tool)

Anything that changes the echo changes the number — ✨ gloss · 💡 light · 📏 height.

The live demo beats the diagram: run a print(left_eye.reflection()) loop, hold the robot, and let students slide surfaces under the eye, tilt it, lift it — the number dances. Establish the active-sensor idea: it brings its own LED, so it mostly ignores the room, but gloss, sensor height, and strong ambient light still shift the echo — which is exactly why every surface gets its own calibration (next slide). reflection() vs color(): today reflection does all the work; color() names colors and gets a cameo here — its real job is Week 4’s notes. Fix it: calibrate, then decide — if / else 🎚️

REFLECT_THRESHOLD = 50              # for THIS surface
while True:

Break it: only two answers 🔥

Two moves only — hard left or hard right. Predict the path.

while True:
    if left_eye.reflection() < THRESHOLD:
        left.run(LINE_SPEED + TURN_HARD)       # on black → yank right
        right.run(LINE_SPEED - TURN_HARD)

Two answers = it can never go straight — it slams left-right forever.

This is bang-bang — the naïve controller. Let them predict, then confirm with the sim two slides on. The diagnosis you want in their words: “it only knows left or right, so it’s always turning.” The bridge to P is exactly the gain quiz from Day 4: what if the correction were proportional to how far off we are, instead of always full-tilt? Keep bang-bang short — it exists only to make P feel like the rescue.

Name it: P — proportional steering 📈

error = left_eye.reflection() - THRESHOLD    # signed: how far off the edge?
left.run(LINE_SPEED + KP * error)            # far → hard steer,
right.run(LINE_SPEED - KP * error)           # near → gentle

error × gain again — tune KP: too small = lazy, too big = the wobble is back.

The “aha” is that they’ve written this line before — twice. It’s literally the D4 self-healing straight with a new error source: swap hub.imu.heading() for the eye’s reflection and the two run lines are identical. Same error × gain, different sensor feeding it. Two failure modes to name so they recognize them on the bench: sluggish (KP too low) drifts wide on curves; twitchy (KP too high) reintroduces the wobble. Start small and adjust. This alone wins “Line King” on gentle lines — but push the speed up on a curve and it starts to overshoot, which is the doorway to the PD stretch. If a student raises the “use motor.dc()” advice here: same answer as the D4 proportional-turn coach note — run() keeps “speed” meaning speed, so KP * error stays a correction in deg/s they can reason about.

Name it: follow_line joins the library 📦

def follow_line(distance_mm, speed=250, kd=0):
    """P follow on the LEFT eye — kd > 0 is the PD stretch."""
    finish = left.angle() + distance_mm * DEG_PER_MM
    last_error = 0
    while left.angle() < finish:
        error = left_eye.reflection() - THRESHOLD
        steer = KP_LINE * error + kd * (error - last_error)
        left.run(speed + steer)
        right.run(speed - steer)
        last_error = error
        wait(10)
    left.stop(); right.stop()

The P loop, bottled — a finish line in degrees makes it a segment. kd is the stretch knob.

Same promotion ritual as every day this week: the loose P loop gets a name and moves into my_lib.py (the gain becomes KP_LINE — the library’s constants block already owns KP for the gyro). Map every part back to what they own: error = P on the eye, the finish line = drive_straight’s DEG_PER_MM trick (follow for a distance — that’s how missions call it), kd * (error - last_error) = the D from the library-day gyro turn, with last_error the one-line memory that makes D possible. The gate is the P version — follow_line(600) with kd=0; the stretch is sliding kd up until the fast curve stops overshooting, which is what wins Speed Demon. State the I-policy plainly and move on: integral fixes a steady offset over a long run; our segments are short, so I is a demo they can watch but never something the gate depends on.

The two-eye job: square up 📐

The second eye’s moment: you hit a junction crooked — sit square before the next move.

def square_on_line(speed=160):
    """Stop each wheel as ITS OWN eye reaches the line."""
    left_done = right_done = False
    left.run(speed); right.run(speed)
    while not (left_done and right_done):
        if not left_done and left_eye.reflection() < THRESHOLD:
            left.hold(); left_done = True      # park left wheel
        if not right_done and right_eye.reflection() < THRESHOLD:
            right.hold(); right_done = True    # park right wheel
        wait(5)

Each eye stops its own wheel — the robot pivots square. That’s why it carries two.

Answer the port question head-on: two color sensors fill two of the six hub ports (B and F), and following needs only one — so the second eye earns its seat here, squaring. On the real mat this kills accumulated drift in a fraction of a second, gold going into the chained runs of Week 3. (The same right eye later reads note colors in Week 4 — but never steers.)

D. Compete & Gate

Today’s showdown: Line King 👑 — and for the fast squads, Speed Demon. 🏁

Line King 👑 / Speed Demon 🏁

The challenge

Line King — hold the curved line for 30 s: accuracy + speed. Speed Demon (fast squads) — fastest run that never leaves the line.

The winning recipe

  • Calibrate on the mat you run on
  • Tune KP_LINE, then chase speed
  • Square at the junction to reset
  • (stretch) kd kills the curve overshoot

Where squads lose it

  • Threshold from last week’s tape, not this mat 😬
  • KP so high the straight wobbles
  • Skipping the square → drift compounds

Line King rewards a clean, honest tune over raw speed; Speed Demon is the stretch for squads whose P was solid early and who took the kd upgrade. Coach’s eye is on the tuning conversation, not the trophy: can they explain what the error means, what KP_LINE did, and why the threshold had to be re-measured on this mat? That explanation IS the gate (kd talk is stretch credit, not a requirement). Keep the taped junction handy so squaring gets used under pressure, not just demoed.
