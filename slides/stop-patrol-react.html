Point out why this beats an ultrasonic sensor even if we had a free port: a touch via stall is repeatable to the millimetre and never lies about a wall it can “see through.”

Fix it: stall = touch, then back off 🧱

APPROACH_SPEED = 150         # deg/s — gentle, not a crash
right.run(APPROACH_SPEED)    # right rolls along…
left.run_until_stalled(APPROACH_SPEED,
                       duty_limit=50)   # …until LEFT jams
right.stop()
hub.speaker.beep()           # "I arrived"
drive_straight(-BACKOFF_MM)         # measured, repeatable

🤖 Robots out. Build it, tune BACKOFF_MM against the board — this is today’s competition in seven lines.

Gentle touch · wait for the jam · back off a measured BACKOFF_MM.

Stress the gentleness knobs: a low approach speed (≈150 deg/s) and duty_limit=50 — the duty limit caps how hard the motor is allowed to push, so the reading is crisp and the LEGO survives. Driving straight into a wall jams both wheels, so watching ONE motor is enough — run_until_stalled blocks on the left while the right just rolls. Line 6 calls their own drive_straight with a negative distance — the challenge parked at the library day is now urgent: flip the while comparison (or mirror it for negatives) in my_lib.py, two-minute fix, big payoff. Back-off is measured, not guessed — BACKOFF_MM = 50 ends exactly 5 cm off the wall, every run. (The pro equivalent, DriveBase.stalled(), arrives with the D8 reveal.)

The point is emotional before it’s technical: “your code was right — it just stopped too soon.” Loops are not a new trick from scratch; they already met while and for on Day 3’s square. Today those loops wrap a sensed decision instead of a fixed count of sides. (The animated one-shot-vs-patrol comparison lives in the 🏠 appendix.)

Name it: the patrol loop 🔁

hub.imu.reset_heading(0)
for lap in range(PATROL_LAPS):          # counted repeat = laps
    left.run(PATROL_SPEED); right.run(PATROL_SPEED)
    while not left.stalled():           # repeat until the wall
        if left_eye.reflection() < REFLECT_THRESHOLD:
            hub.speaker.beep(880)       # react, don't stop
        wait(10)
    left.stop(); right.stop()
    drive_straight(-BACKOFF_MM)                # measured back-off
    turn_to(90 * (lap + 1))             # absolute appointment

Two loops nested — for counts laps, while waits for the wall, the if reacts on the move.

Read the shape aloud: outer for = “do this lap N times,” inner while not left.stalled() = “keep rolling until you feel the wall,” inner if = “and if you cross a staff line on the way, sing out” — yesterday’s calibrated threshold, reused as-is. The if living inside the while is the subtle win — reaction happens continuously during motion, not as a stop-and-check. (That’s also why this loop uses left.stalled() rather than the blocking run_until_stalled from this morning — the robot needs to keep checking things while it rolls.) turn_to(90 * (lap + 1)) is the library day’s absolute-appointment trick doing its job silently: corner 1 aims at 90, corner 2 at 180 — a bumped corner can’t poison the next one. This program is the literal answer key for Gentle Giant’s loop bonus. With one board and no table borders yet, patrol two “walls” by moving the board between reps or using a wall + heavy box pair — the loop doesn’t care.

D. Bottle It: Two New Verbs

Bottle today’s tricks into my_lib.py — then read the scoresheet like a pro. 📦

drive_to_wall and wall_square join the library 📦

def drive_to_wall(speed=150):
    """Roll gently until the wheels feel the wall."""
    right.run(speed)
    left.run_until_stalled(speed, duty_limit=50)
    right.stop()

def wall_square(heading=0):
    """Press BOTH wheels flat on the wall → truth restored."""
    left.run(120); right.run(120)
    while not (left.stalled() and right.stalled()):
        wait(10)
    left.stop(); right.stop()
    hub.imu.reset_heading(heading)   # the wall never lies
    drive_straight(-BACKOFF_MM)

Both wheels flat on the wall → heading known again. The wall never lies.

Same promotion ritual as the library day: the morning’s loose fix-it lines get names and move into my_lib.py, right under drive_straight and turn_to. Walk wall_square slowly — it’s the first verb whose output is trust, not motion: waiting for BOTH motors to stall means the bumper is flat on the wall, so the physical world just told you your heading exactly; reset_heading(heading) writes that truth back into the gyro, and the measured back-off leaves you at a known distance too. Gyro drift accumulates over a 2-minute run — squaring is how Weeks 3–4 missions reset it between tasks, which is why this humble verb outlives almost everything else. Demo: spin the robot a few times by hand (drift injected), wall_square, then a turn_to(90) — still true. Table note (decision #23): until the bordered table exists, wall_square trains on the fixed board; when the table lands we re-validate it at the true border positions — that’s the table-arrival checkpoint.
